# Library Setup

Installing ArchUI into your existing React project is quite easy and should not take you more than just a few minutes to setup. Follow the steps below and you'll be ready to start using and editing your favorite components.

## **Step 1: Required Packages**

Make sure you are using **React 19+** and **Tailwind CSS 4+** by heading over to your `package.json` file and checking under "dependencies".
If you don't already have the required versions listed, you'll need to update or install them before proceeding.

<Source
  dark
  language="json"
  code={`"dependencies": {
    "react": "^19.1.1",
    "react-dom": "^19.1.1",
    "tailwindcss": "^4.1.14"
}`}
/>

## **Step 2: Tailwind Setup**

Since ArchUI is built on top of Tailwind CSS, you'll need to have Tailwind 4+ set up in your project. There are two common ways to add Tailwind CSS:

#### **Option A:** Automatic Setup

Head over to [Tailwind CSS](https://tailwindcss.com/docs/installation) and follow the installation steps that apply to your React project. In this example, we will be using Vite, so the Vite plugin will be required.

<Source dark language="bash" code={`npm install @tailwindcss/vite`} />

#### **Option B:** Manual Setup

You can also add set up Tailwind manually by creating the following files: `tailwind.config.js`, `postcss.config.js`, and `vite.config.ts` (or the build tool of your choice), then paste the following code inside each file:

**tailwind.config.js**

<Source
  dark
  language="js"
  code={`/** @type {import('tailwindcss').Config} */
export default {
  content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"],
  theme: {
    extend: {},
  },
  plugins: [],
};`}
/>

**vite.config.ts**

<Source dark language="ts" code={`import { defineConfig } from "vite";
import react from "@vitejs/plugin-react-swc";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
plugins: [react(), tailwindcss()],
css: {
postcss: "./postcss.config.js",
},
});`} />

**postcss.config.js**

<Source
  dark
  language="js"
  code={`export default {
  plugins: {
    "@tailwindcss/postcss": {},
    autoprefixer: {},
  },
};`}
/>

## **Step 3: Install ArchUI**

Open up the CLI and install the ArchUI Library with the following command:

<Source dark language="bash" code={`npm i @devtoti/archui-lib`} />

## **Step 4: Setup ArchUI into your project**

Import the library's main CSS file directly inside `index.css`, so that your project can have access to ArchUI's design system.
Make sure to import Tailwind CSS first, and then ArchUI's styles.

<Source
  dark
  language="css"
  code={`@import "tailwindcss";
@import "@devtoti/archui-lib/styles";`}
/>

## **Step 5: Wrap your app in the ThemeProvider**

Wrap your App in the ThemeProvider to ensure that your components will be styled correctly. You can also pass in the initial theme and theme type to the ThemeProvider:

<Source
  dark
  language="tsx"
  code={`import { ThemeProvider } from "@devtoti/archui-lib";

return (
  <ThemeProvider theme="ionic" themeType="light">
    <App />
  </ThemeProvider>
);`}
/>

## **Step 6: Import Your First Component**

Now that you have configured Tailwind, ArchUI and your ThemeProvider, you can start using your favorite components by importing them into your React files. Additionally, you can import the Component's Type Props and adapt them to your needs:

<Source
  dark
  language="tsx"
  code={`import { Button } from "@devtoti/archui-lib";
import type { ButtonProps } from "@devtoti/archui-lib";

function App() {
return (
  <AppContent>
    <Button
          variant="primary"
          size="md"
          onClick={handleRegister}
          label="Welcome to ArchUI!"
        />
  </AppContent>
);
}
`} />

## **Step 6: Delete default styles that might conflict with ArchUI**

Delete default styles that might conflict with ArchUI. For example, if you are using Tailwind, you can delete the content of the following files:

- `App.css`
- `index.css`
- `styles.css`

## **Step 7: Browse Our Components**

You can now start exploring our [_repertoire_](/docs/01-components-alert--docs) of components library, or use our experimental Drag-and-Drop Playground to get a better grasp of how things work.
